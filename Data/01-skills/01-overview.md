# Skills 개요

## Skills란 무엇인가?

**Skills**는 Claude가 특정 작업을 수행하는 방법을 알려주는 Markdown 파일입니다. 사용자가 Skill의 목적과 관련된 질문을 하면, Claude가 자동으로 해당 Skill을 적용합니다.

### 핵심 특징

| 특징 | 설명 |
|------|------|
| **모델 호출 방식** | 사용자가 명시적으로 호출할 필요 없이, Claude가 상황에 맞게 자동 선택 |
| **자동 발견** | Skill의 description과 사용자 요청 간 의미론적 매칭 |
| **점진적 공개** | 메타데이터만 미리 로드, 상세 내용은 필요 시 읽음 |
| **컨텍스트 효율성** | 보조 파일은 Claude가 실제로 읽기 전까지 토큰 소비 없음 |

## Skills vs 다른 커스터마이징 옵션

| 기능 | 사용 시점 | 실행 방식 |
|------|----------|----------|
| **Skills** | Claude에게 전문 지식 부여 | Claude가 관련 시 자동 선택 |
| **Slash Commands** (`/command`) | 빠른 재사용 프롬프트 | `/command` 명시적 입력 |
| **CLAUDE.md** | 프로젝트 전역 지침 설정 | 모든 대화에서 로드 |
| **Subagents** | 별도 컨텍스트에 위임 | Claude가 위임하거나 사용자가 호출 |
| **Hooks** | 도구 이벤트에 스크립트 실행 | 특정 도구 이벤트 시 발동 |
| **MCP Servers** | 외부 도구 연결 | Claude가 필요 시 MCP 도구 호출 |

## 실제 활용 사례

### 1. 팀 표준 코드 리뷰
```
사용자: "이 PR을 리뷰해줘"
Claude: [code-review Skill 자동 적용] → 팀의 코드 리뷰 체크리스트에 따라 리뷰 수행
```

### 2. 커밋 메시지 생성
```
사용자: "커밋 메시지 작성해줘"
Claude: [commit-helper Skill 자동 적용] → 회사 컨벤션에 맞는 커밋 메시지 생성
```

### 3. 데이터베이스 스키마 쿼리
```
사용자: "사용자 테이블에서 최근 가입자를 찾아줘"
Claude: [bigquery-analysis Skill 자동 적용] → 정확한 스키마와 쿼리 패턴으로 응답
```

### 4. PDF 처리
```
사용자: "이 PDF에서 텍스트를 추출해줘"
Claude: [pdf-processing Skill 자동 적용] → pdfplumber를 사용한 추출 코드 제공
```

## Skills의 작동 원리

```
┌─────────────────────────────────────────────────────────────┐
│                    사용자 요청                               │
│              "PDF에서 데이터를 추출해줘"                      │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                 Claude의 Skill 매칭                          │
│                                                             │
│  1. 로드된 Skill들의 description 검토                        │
│  2. "PDF", "추출" 키워드와 매칭되는 Skill 탐색                │
│  3. pdf-processing Skill 발견                                │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                 Skill 적용 요청                              │
│                                                             │
│  "pdf-processing Skill을 사용하겠습니다. 허용하시겠습니까?"   │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│               SKILL.md 전체 내용 로드                        │
│                                                             │
│  - 상세 지침 읽기                                            │
│  - 필요 시 참조 파일(FORMS.md, REFERENCE.md) 추가 로드       │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                  작업 수행 및 응답                           │
└─────────────────────────────────────────────────────────────┘
```

## Skill 파일 위치와 스코프

| 위치 | 경로 | 가시성 |
|------|------|--------|
| **Personal** | `~/.claude/skills/` | 모든 프로젝트에서 본인만 사용 |
| **Project** | `.claude/skills/` | 저장소의 모든 팀원이 사용 |
| **Plugin** | 플러그인 디렉토리 내 `skills/` | 플러그인 설치자 모두 사용 |
| **Enterprise** | 관리자 설정 | 조직 전체 |

## 언제 Skill을 사용해야 하는가?

### Skill이 적합한 경우
- Claude에게 **도메인 특화 지식**을 부여하고 싶을 때
- 여러 대화에서 **반복적으로 같은 컨텍스트**를 제공할 때
- 팀 전체가 **일관된 방식**으로 작업해야 할 때
- **복잡한 워크플로우**를 표준화하고 싶을 때

### Slash Command가 더 적합한 경우
- **빠르고 단순한** 반복 작업
- 명시적으로 **사용자가 호출**해야 하는 기능
- 단일 프롬프트로 완료되는 작업
