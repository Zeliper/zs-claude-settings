# CLAUDE.md 개요

## CLAUDE.md란 무엇인가?

CLAUDE.md는 Claude Code에 프로젝트 맥락과 작업 지침을 제공하는 마크다운 파일입니다. Claude Code 세션이 시작될 때 자동으로 로드되어, Claude가 프로젝트의 구조, 컨벤션, 워크플로우를 이해하고 일관된 방식으로 작업할 수 있도록 합니다.

### 핵심 특징

| 특징 | 설명 |
|------|------|
| **자동 로드** | 세션 시작 시 자동으로 읽어들임 |
| **계층적 적용** | 프로젝트, 사용자, 엔터프라이즈 수준 지원 |
| **마크다운 형식** | 읽기 쉽고 버전 관리 용이 |
| **즉시 반영** | 파일 수정 시 다음 세션부터 적용 |

---

## CLAUDE.md 파일 위치와 스코프

CLAUDE.md는 여러 위치에 배치할 수 있으며, 각 위치에 따라 적용 범위가 달라집니다.

| 위치 | 스코프 | 우선순위 | 용도 |
|------|--------|---------|------|
| `./CLAUDE.md` | 현재 프로젝트 | 높음 | 프로젝트별 규칙과 컨벤션 |
| `~/.claude/CLAUDE.md` | 사용자 전체 | 중간 | 개인 작업 스타일과 선호도 |
| 엔터프라이즈 설정 | 조직 전체 | 낮음 | 팀/회사 표준 가이드라인 |

### 우선순위 병합 규칙

여러 위치에 CLAUDE.md가 존재할 경우, 다음 순서로 병합됩니다:

```
┌─────────────────────────────────────────────────┐
│  1. 프로젝트 CLAUDE.md (./CLAUDE.md)            │ ← 최우선
├─────────────────────────────────────────────────┤
│  2. 사용자 CLAUDE.md (~/.claude/CLAUDE.md)      │
├─────────────────────────────────────────────────┤
│  3. 엔터프라이즈 설정                            │ ← 기본값
└─────────────────────────────────────────────────┘
```

## CLAUDE.md vs 다른 설정 방식 비교

Claude Code에서 동작을 커스터마이징하는 여러 방법이 있습니다. 상황에 맞는 방식을 선택하세요.

| 방식 | 용도 | 지속성 | 공유 가능 |
|------|------|--------|----------|
| **CLAUDE.md** | 프로젝트 맥락과 규칙 전달 | 영구적 | Git으로 공유 |
| **Skills** | 반복 작업 자동화 | 영구적 | 파일로 공유 |
| **System Prompt** | 세션별 행동 지시 | 세션 한정 | 수동 복사 |
| **대화 중 지시** | 임시 요청 | 대화 한정 | 공유 불가 |

### 언제 CLAUDE.md를 사용하는가?

**적합한 경우:**
- 프로젝트의 기술 스택과 구조를 설명할 때
- 코딩 컨벤션과 스타일 가이드를 정의할 때
- 팀 전체가 일관된 작업 방식을 유지해야 할 때
- 반복적으로 설명해야 하는 맥락이 있을 때

**부적합한 경우:**
- 일회성 작업 지시 → 대화 중 직접 요청
- 복잡한 반복 작업 → Skills 사용
- 외부 도구 연동 → MCP 서버 구성

## CLAUDE.md의 작동 원리

Claude Code 세션이 시작되면 다음 순서로 CLAUDE.md를 처리합니다:

```
┌────────────────┐
│ 세션 시작       │
└───────┬────────┘
        ▼
┌────────────────────────────────────┐
│ 1. 엔터프라이즈 설정 로드 (있는 경우) │
└───────┬────────────────────────────┘
        ▼
┌────────────────────────────────────┐
│ 2. ~/.claude/CLAUDE.md 로드         │
└───────┬────────────────────────────┘
        ▼
┌────────────────────────────────────┐
│ 3. ./CLAUDE.md 로드                 │
└───────┬────────────────────────────┘
        ▼
┌────────────────────────────────────┐
│ 4. 병합된 맥락으로 Claude 초기화     │
└───────┬────────────────────────────┘
        ▼
┌────────────────┐
│ 대화 시작       │
└────────────────┘
```

## 실제 활용 시나리오

### 시나리오 1: 새 팀원 온보딩

프로젝트에 새 팀원이 합류했을 때, CLAUDE.md가 있으면 Claude가 프로젝트 컨벤션을 자동으로 이해합니다.

```markdown
# Project: E-commerce Platform

## 기술 스택
- Frontend: Next.js 14 + TypeScript
- Backend: Node.js + Express
- Database: PostgreSQL + Prisma

## 코딩 컨벤션
- 컴포넌트는 함수형으로 작성
- CSS는 Tailwind CSS 사용
- API 응답은 항상 { success, data, error } 형태
```
